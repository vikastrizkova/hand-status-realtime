# Hand Status Realtime — мой проект

**Живой сайт (фронтенд):** [https://vikastrizkova.github.io/hand-status-realtime/](https://vikastrizkova.github.io/hand-status-realtime/)

## Как устроен проект

- **Фронтенд** (React + Vite) размещён на **GitHub Pages** — показывает текущее состояние («В руке» / «Не в руке») и автоматически обновляется без перезагрузки.
- **Бэкенд** (Node.js + Express + Socket.IO) работает на **Render.com** — управляет состоянием и рассылает обновления всем подключённым клиентам через WebSocket.

## Управление состоянием

Вы можете менять состояние **двумя способами** (работают одновременно).

### 1️⃣ Кнопка на самом сайте
На странице есть кнопка. При её нажатии состояние меняется мгновенно, страница не перезагружается.

### 2️⃣ Изменение через URL (GET-запрос к бэкенду)
Просто перейдите по одной из ссылок ниже — состояние на сайте обновится автоматически (даже если сайт открыт в другой вкладке).

- **Установить «В руке» (true):**  
  [https://hand-status-realtime.onrender.com/set-state?inHand=true](https://hand-status-realtime.onrender.com/set-state?inHand=true)

- **Установить «Не в руке» (false):**  
  [https://hand-status-realtime.onrender.com/set-state?inHand=false](https://hand-status-realtime.onrender.com/set-state?inHand=false)

После перехода по ссылке откройте (или обновите) сайт по адресу выше — надпись изменится без нажатия кнопки.


## Важное примечание для проверяющего

Бесплатный тариф на Render.com «засыпает» через 15 минут бездействия. При первом обращении к сайту или управляющим ссылкам возможна задержка 30–50 секунд — это нормально. После «пробуждения» всё работает быстро и стабильно.
